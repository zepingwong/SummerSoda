Windows ssh-add 报错 Error connecting to agent No such file or directory

WANG Zeping 2020/2/11 WindowsSSH

# 问题描述

# 环境

  • 操作系统：Windows 10/11
  • 终端：Windows Terminal、Windows PowerShell

# 问题

Windows环境下执行

ssh-add ~/.ssh/id_rsa
1

报错如下

Error connecting to agent: No such file or directory
1

# 问题描述

# 环境

  • 操作系统：Windows 10/11
  • 终端：Windows Terminal、Windows PowerShell

# 问题

Windows环境下执行

ssh-add ~/.ssh/id_rsa
1

报错如下

Error connecting to agent: No such file or directory
1

# 解决步骤

# 1. 启动PowerShell

管理员身份启动 「Windows PowerShell」。

# 2. 检查服务

运行以下指令，检查 ssh-agent 服务是否启动成功。

get-service ssh*
1

若输出如下，显示 Status 结果为Stopped，则表示服务未启动。

Status   Name               DisplayName
------   ----               -----------
Stopped  ssh-agent          OpenSSH Authentication Agent
1
2
3

# 3. 启动服务

使用以下两条指令启动ssh-agent服务

Set-Service -Name ssh-agent -StartupType Manual
Start-Service ssh-agent
1
2

再次使用 get-service ssh* 查看服务是否启动，如下图所示显示 Status 为 Running 则表示， ssh-agent 服务已启动。

# 4. 检查

运行以下指令查看 ssh-agent 已经添加的秘钥：

ssh-add -l
1

如果未未添加过秘钥，则输出 The agent has no identities
